# Criterios De Calidad

## Seguridad

- No leer ni editar secretos.
- No tocar datos reales desde pruebas.
- No ejecutar comandos destructivos.
- No publicar, desplegar ni hacer merge desde OpenCode.

## Producto

- El cambio debe responder al pedido real.
- Si cambia una regla de negocio, pedir confirmación humana.
- Evitar soluciones enormes cuando alcanza con una corrección chica.

## Frontend

- Mobile-first.
- Accesible: contraste, textos claros, controles entendibles y navegación usable.
- Diseño industrial: sobrio, robusto y enfocado en la tarea.

## Backend Y Datos

- Validar entradas.
- Proteger permisos.
- Evitar efectos secundarios sobre datos existentes.
- Verificar localmente cuando sea posible.

## Comunicación

- Explicar qué cambió.
- Explicar cómo se verificó.
- Decir qué queda pendiente o qué riesgo no se pudo cubrir.
