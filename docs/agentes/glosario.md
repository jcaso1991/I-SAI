# Glosario

- API: puerta de comunicación entre frontend y backend. Como una ventanilla donde la app pide datos.
- Backend: parte del sistema que guarda datos y aplica reglas. Como la oficina interna.
- Build: paquete final de la app para publicar o distribuir.
- Commit: punto de guardado en Git con una descripción.
- Draft PR: pedido de revisión en borrador, todavía no listo para mezclar.
- Endpoint: dirección específica de la API para pedir o enviar datos.
- Frontend: parte visible de la app que usa la persona.
- JWT: credencial digital para saber quién está usando el sistema.
- Mongo/Motor: base de datos y herramienta para hablar con ella desde Python.
- Pydantic: validación de datos en Python; funciona como un formulario con campos obligatorios.
- Snapshot: foto de seguridad que permite volver atrás con `/undo`.
